# CAN 报文显示、收发与信号曲线界面开发需求

## 1. 开发目标

开发一个桌面端 CAN/CAN FD 调试软件界面，界面风格参考 ZLG ZXDoc 这类汽车总线分析工具。

本阶段重点不是实现完整汽车总线工具链，而是优先实现以下界面与交互：

1. CAN 通道管理界面；

2. CAN 实时报文接收显示界面；

3. CAN 报文发送配置界面；

4. DBC 报文/信号解析界面；

5. 根据具体信号绘制实时曲线；

6. 报文窗口、信号窗口、曲线窗口之间的同步联动；

7. 记录、回放、过滤、统计等基础辅助功能。

软件界面应采用工程化、多窗口、表格化、可停靠面板的风格，接近专业汽车总线分析软件，而不是普通简易串口助手风格。

主界面采用类似工业调试软件的多区域布局。

## 2.1 主窗口结构

主窗口建议分为以下区域：

```text

├──────────────────────────────────────────────────────────────┤
│ 顶部工具栏：连接设备 / 启动测量 / 停止 / 记录 / 回放 / 清空 / 过滤 │
├───────────────┬──────────────────────────────────────────────┤
│ 左侧工程树     │ 中央主区域：CAN 报文显示窗口                    │
│               │                                              │
│ - 硬件设备     │                                              │
│ - CAN 通道     │                                              │
│ - DBC 数据库   │                                              │
│ - 发送列表     │                                              │
│ - 曲线列表     │                                              │
├───────────────┴──────────────────────────────────────────────┤

└──────────────────────────────────────────────────────────────┘
```

## 2.2 窗口停靠要求

界面必须支持多个可停靠窗口，至少包括：

| 窗口名称     | 默认位置    | 说明                    |
| -------- | ------- | --------------------- |
| 工程树窗口    | 左侧      | 显示设备、通道、数据库、发送任务、曲线配置 |
| CAN 报文窗口 | 中央      | 实时显示 Rx/Tx 报文         |
| 信号解析窗口   | 底部      | 显示选中报文的 DBC 信号        |
| 发送报文窗口   | 底部 Tab  | 配置单次发送和周期发送           |
| 曲线窗口     | 底部或独立窗口 | 根据选中信号绘制实时曲线          |
| 统计窗口     | 底部 Tab  | 显示总线负载、帧率、错误帧、ID 周期   |
| 日志窗口     | 底部 Tab  | 显示软件运行日志              |

## 2.3 Tab 风格

中央和底部区域应支持 Tab 页。

例如底部 Tab：

```text
[信号解析] [发送报文] [曲线] [统计信息] [运行日志]
```

每个 Tab 可以被拖拽为独立浮动窗口，也可以重新停靠。

如果第一版开发复杂，可先固定布局，但代码结构需要预留 Docking Layout 扩展能力。

# 6. CAN 报文显示窗口

CAN 报文显示窗口是主界面核心区域，采用高性能表格显示。

## 6.1 报文表格字段

表格字段按以下顺序显示：

| 列名      | 示例                      | 说明                    |
| ------- | ----------------------- | --------------------- |
| No.     | 1                       | 报文序号                  |
| Time    | 12.345678               | 相对时间戳，单位秒             |
| Delta   | 0.010000                | 与同 ID 上一帧时间差          |
| Ch      | CAN1                    | 通道                    |
| Dir     | Rx                      | Rx 或 Tx               |
| ID      | 0x180                   | CAN ID                |
| Name    | BMS_Status              | DBC 报文名               |
| Type    | Std                     | Std 或 Ext             |
| Frame   | Data                    | Data / Remote / Error |
| FD      | No                      | 是否 CAN FD             |
| BRS     | No                      | 是否开启 BRS              |
| DLC     | 8                       | DLC                   |
| Len     | 8                       | 实际数据长度                |
| Data    | 01 02 03 04 05 06 07 08 | 数据区                   |
| Cycle   | 100 ms                  | DBC 周期或统计周期           |
| Count   | 1200                    | 当前 ID 收到帧数            |
| Comment | Normal                  | 状态备注                  |

## 6.2 表格视觉样式

表格要求：

- 表头固定；

- 行高 24px 到 28px；

- 字体使用等宽字体，例如 Consolas；

- CAN ID、Data 使用等宽字体；

- 支持横向滚动；

- 支持纵向虚拟滚动；

- 支持列宽拖拽；

- 支持列显示/隐藏；

- 支持点击列头排序；

- 支持多选行；

- 支持复制选中行。

## 6.3 报文显示模式

报文窗口必须支持两种模式。

### 6.3.1 Trace 模式

也叫滚动模式。

每收到一帧就在表格新增一行。

适合观察完整报文流。

特点：

- 按时间顺序追加；

- 支持自动滚动到底部；

- 用户滚动查看历史时自动暂停跟随；

- 点击“跟随最新”后恢复自动滚动；

- 可配置最大保留行数，默认 100000 行。

### 6.3.2 Overwrite 模式

也叫固定 ID 刷新模式。

相同通道、相同方向、相同 CAN ID 只显示一行，新报文刷新该行。

适合观察当前网络状态。

特点：

- 每个 ID 一行；

- Count 字段持续累加；

- Time 显示最近更新时间；

- Delta 显示最近一次周期；

- Data 显示最新数据；

- 变化的字节高亮；

- 超时 ID 标记为橙色。

## 6.4 Trace / Overwrite 切换位置

在报文窗口顶部增加小工具条：

```text
[Trace 模式] [Overwrite 模式] [暂停] [清空] [过滤] [导出] [自动滚动]
```

按钮要求：

- 当前模式高亮；

- 切换模式不清空缓存；

- 清空按钮只清空界面缓存，不影响记录文件。

## 6.5 报文颜色规则

默认颜色规则如下：

| 报文类型      | 显示样式                           |
| --------- | ------------------------------ |
| Rx 普通报文   | 白色背景，黑色文字                      |
| Tx 报文     | 浅蓝色背景                          |
| 错误帧       | 浅红色背景，红色文字                     |
| 远程帧       | 浅紫色背景                          |
| CAN FD 报文 | ID 或 FD 列加蓝色标识                 |
| 新出现 ID    | 行背景短暂浅绿色闪烁                     |
| 数据变化字节    | Data 中变化字节黄色高亮                 |
| 超时 ID     | 行背景浅橙色                         |
| DBC 未匹配   | Name 显示为空或灰色 Unmapped          |
| 信号越界      | Comment 显示 Signal Out Of Range |

颜色应集中在主题配置文件中，方便后续修改。

## 6.6 数据变化高亮规则

对于同一通道、同一 ID：

1. 保存上一帧 Data；

2. 新帧到来时逐字节比较；

3. 发生变化的字节在 Data 单元格中高亮；

4. 高亮持续 500ms；

5. 如果多字节变化，只高亮变化字节，不高亮整个 Data 单元格。

显示示例：

```text
上一帧：01 02 03 04 05 06 07 08
当前帧：01 02 03 04 05 09 07 08
显示：  01 02 03 04 05 [09] 07 08
```

## 6.7 报文右键菜单

# 8. 曲线窗口需求

曲线窗口用于根据 DBC 信号绘制实时曲线。

## 8.1 曲线窗口入口

用户可以通过以下方式打开曲线：

1. 顶部菜单：曲线 → 新建曲线窗口；

2. 工具栏：点击“添加曲线”；

3. 报文右键：选择信号添加到曲线；

4. 信号解析窗口右键：添加到曲线；

5. 工程树中对某个 DBC 信号右键：添加到曲线；

6. 双击信号解析窗口中的信号。

## 8.2 曲线窗口整体布局

曲线窗口采用左侧信号列表 + 右侧曲线图的布局。

```text
├───────────────────┬──────────────────────────────────────────┤
│ 信号列表           │ 曲线绘图区                               │
│ □ BMS_SOC          │                                          │
│ □ BMS_Voltage      │       实时折线图                          │
│ □ BMS_Current      │                                          │
│                   │                                          │
├───────────────────┴──────────────────────────────────────────┤
│ 状态栏：当前时间 / 鼠标位置 / 游标值 / 采样点数量               │
└──────────────────────────────────────────────────────────────┘
```

## 8.3 曲线工具栏

曲线窗口顶部工具栏包含：

```text
[开始] [暂停] [清空] [自动缩放] [固定 Y 轴] [游标] [缩放] [平移] [导出]
```

## 8.5 曲线绘图区要求

曲线绘图区应满足：

- X 轴为时间；

- Y 轴为信号物理值；

- 支持多信号同时显示；

- 支持不同单位信号同时显示；

- 支持多 Y 轴或归一化显示；

- 支持鼠标滚轮缩放；

- 支持鼠标拖动平移；

- 支持十字游标；

- 支持显示当前点时间和值；

- 支持暂停后查看历史；

- 支持自动跟随最新数据；

- 支持隐藏/显示某条曲线；

- 支持清空曲线。

## 8.6 X 轴时间显示

X 轴支持两种模式：

### 相对时间模式

从开始测量开始计时。

示例：

```text
0s, 5s, 10s, 15s, 20s
```

### 绝对时间模式

显示真实时间。

示例：

```text
14:32:10.100, 14:32:11.100, 14:32:12.100
```

界面上提供切换按钮：

```text
时间模式：[相对时间] [绝对时间]
```

## 8.7 Y 轴显示

Y 轴默认自动缩放。

每个信号可以配置：

| 配置项       | 说明                  |
| --------- | ------------------- |
| 自动 Y 轴    | 根据数据自动缩放            |
| 固定 Y 轴    | 用户设置 Min/Max        |
| 使用 DBC 范围 | 使用 DBC 中定义的 Min/Max |
| 独立 Y 轴    | 每个信号独立 Y 轴          |
| 共享 Y 轴    | 相同单位信号共享 Y 轴        |
| 归一化       | 所有信号归一化到 0~100%     |

## 8.8 添加信号到曲线的流程

当用户对信号选择“添加到曲线”时：

```text
用户选中 CAN 报文
    ↓
信号解析窗口显示该报文信号
    ↓
用户右键信号 BMS_SOC
    ↓
点击“添加到曲线”
    ↓
如果曲线窗口未打开，则自动打开
    ↓
左侧信号列表添加 BMS_SOC
    ↓
曲线绘图区开始绘制 BMS_SOC 的实时物理值
```

## 8.9 曲线数据来源

曲线数据来自 DBC 解析后的信号物理值。

每收到一帧 CAN 报文后：

1. 根据 CAN ID 找到 DBC Message；

2. 解码所有 Signal；

3. 如果某个 Signal 已被添加到曲线；

4. 将 timestamp 和 physical value 写入曲线缓存；

5. 曲线窗口按固定刷新频率批量刷新。

## 8.10 曲线刷新性能

要求：

- 曲线数据采集不能阻塞 CAN 接收线程；

- 曲线 UI 不允许每帧刷新；

- 曲线 UI 建议 10Hz 到 30Hz 刷新；

- 每个信号缓存最大点数可配置；

- 默认每个信号保留最近 10000 个点；

- 超过最大点数后采用环形缓冲区；

- 大量数据时应抽稀显示，避免 UI 卡顿。

## 8.11 曲线游标功能

开启游标后：

- 鼠标移动到曲线区域显示十字线；

- 状态栏显示当前时间；

- 显示每条曲线在该时间点附近的值；

- 支持锁定游标；

- 支持两个游标；

- 双游标时显示时间差和值差。

状态栏示例：

```text
Cursor1: 12.345s    BMS_SOC=80.0%    BMS_Voltage=750.2V
Cursor2: 13.345s    Δt=1.000s        ΔSOC=0.2%
```

## 8.12 曲线窗口与报文窗口同步

要求实现窗口同步功能。

### 从报文同步到曲线

用户在 CAN 报文窗口点击某一帧时：

- 曲线窗口游标跳转到该帧 timestamp；

- 曲线窗口显示该时间点附近的信号值；

- 如果当前报文中的信号已在曲线中显示，则该信号曲线高亮 1 秒。

### 从曲线同步到报文

用户在曲线窗口点击某个时间点时：

- CAN 报文窗口自动跳转到最接近该 timestamp 的报文；

- 该报文行高亮；

- 信号解析窗口显示该报文的信号值。

### 从信号同步到曲线

用户在信号解析窗口点击某个信号：

- 如果该信号已在曲线中，曲线高亮；

- 如果未添加，提示“添加到曲线”。

# 16. 曲线刷新逻辑

曲线刷新不允许每收到一帧就刷新 UI。

正确流程：

```text
CAN 接收线程
    ↓
报文进入消息队列
    ↓
DBC 解码服务解析信号
    ↓
如果信号已加入曲线，则写入 ChartDataService 缓存
    ↓
UI 定时器每 50ms 到 100ms 批量刷新曲线
```

## 16.1 刷新要求

- 曲线 UI 刷新周期默认 100ms；

- 可配置为 50ms、100ms、200ms、500ms；

- 后台数据缓存必须完整；

- 暂停曲线只暂停 UI 刷新，不停止数据采集；

- 清空曲线时清空当前显示缓存。

# 18. 给 AI 开发工具的最终提示词

请开发一个参考 ZLG ZXDoc 风格的桌面端 CAN/CAN FD 报文显示与收发软件界面。

重点实现界面，不要只写底层逻辑。

软件主界面必须包含：

1. 顶部菜单栏；

2. 顶部工具栏；

3. 左侧工程树；

4. 中央 CAN 报文显示表格；

5. 底部 Tab 面板，包括信号解析、发送报文、曲线、统计信息、运行日志。

CAN 报文窗口必须支持 Trace 模式和 Overwrite 模式。Trace 模式每帧新增一行，Overwrite 模式相同通道和 CAN ID 只刷新一行。报文表格字段必须包含 Time、Delta、Ch、Dir、ID、Name、Type、FD、BRS、DLC、Len、Data、Cycle、Count、Comment。Data 字段必须使用等宽字体，变化字节需要高亮显示。

加载 DBC 后，报文窗口需要显示报文名称，点击报文后，底部信号解析窗口显示该报文的所有信号，包括 Signal、Raw、Physical、Unit、Min、Max、StartBit、Length、ByteOrder、Factor、Offset、Status。

曲线窗口必须支持根据具体 DBC 信号绘制实时曲线。用户可以从信号解析窗口右键或双击某个信号，将该信号添加到曲线。曲线窗口左侧显示信号列表，右侧显示实时折线图。曲线支持开始、暂停、清空、自动缩放、游标、缩放、平移、导出 CSV。X 轴为时间，Y 轴为物理值。支持多个信号同时显示，支持不同单位信号使用独立 Y 轴或归一化显示。

报文窗口、信号解析窗口、曲线窗口之间必须联动。点击报文时，信号解析窗口刷新，曲线游标跳转到对应时间。点击曲线时间点时，报文窗口跳转到最接近该时间的报文行。

发送窗口必须支持单次发送和周期发送。发送表格字段包括 Enable、Name、Ch、ID、Name(DBC)、Type、FD、BRS、DLC、Data、Mode、Period、Count、Sent、Status。Data 支持十六进制编辑和合法性校验。支持从接收报文右键添加到发送窗口。支持从 DBC 报文创建发送任务，并通过编辑信号物理值自动编码 Data。
