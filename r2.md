# CAN 报文显示窗口详细需求

## 1. 窗口定位

CAN 报文显示窗口是软件中央主区域的核心窗口，用于实时显示 CAN/CAN FD 报文流，并支持用户对报文进行排序、筛选、过滤、搜索、分组、复制、导出、添加到发送窗口、查看 DBC 解析和跳转曲线等操作。

该窗口风格参考专业 CAN 工具软件，应具备高信息密度、高性能刷新、工程化筛选和快速定位能力。

---

# 2. 窗口整体布局

CAN 报文显示窗口由以下区域组成：

```text
┌────────────────────────────────────────────────────────────────────┐
│ 窗口标题栏：CAN 报文                                                │
├────────────────────────────────────────────────────────────────────┤
│ 快捷工具条：Trace / Overwrite / 暂停 / 清空 / 自动滚动 / 导出          │
├────────────────────────────────────────────────────────────────────┤
│ 快速过滤栏：Ch / Dir / ID / Name / Data / DBC / Error / 搜索           │
├────────────────────────────────────────────────────────────────────┤
│ 报文表格区域                                                        │
├────────────────────────────────────────────────────────────────────┤
│ 底部状态栏：显示帧数 / 过滤后帧数 / 当前选中 / Rx / Tx / Error / Load   │
└────────────────────────────────────────────────────────────────────┘
```

# 3. 顶部快捷工具条

## 3.1 工具条按钮

报文窗口顶部应提供以下按钮：

| 按钮           | 类型            | 功能               |
| ------------ | ------------- | ---------------- |
| Trace 模式     | Toggle Button | 每帧新增一行显示         |
| Overwrite 模式 | Toggle Button | 相同通道、方向、ID 只刷新一行 |
| 暂停显示         | Toggle Button | 暂停 UI 刷新，不停止底层接收 |
| 跟随最新         | Toggle Button | 自动滚动到最新报文        |
| 清空窗口         | Button        | 清空当前显示缓存         |
| 快照           | Button        | 将当前显示内容冻结为快照     |
| 导出           | Button        | 导出当前显示或选中报文      |
| 列设置          | Button        | 设置显示/隐藏列         |
| 高级过滤         | Button        | 打开高级过滤弹窗         |

## 3.2 工具条状态

工具条右侧显示当前状态：

```text
模式：Trace    显示：运行中    自动滚动：开    过滤：已启用    行数：128,000
```

## 3.3 暂停显示逻辑

点击“暂停显示”后：

- UI 表格停止追加或刷新；

- 后台接收线程继续接收报文；

- 记录功能继续写入文件；

- 统计功能继续更新；

- 状态栏显示“显示已暂停，后台仍在接收”；

- 再次点击后，UI 继续从最新缓存刷新；

- 暂停期间允许用户排序、筛选、复制和导出当前显示内容。

# 4. 快速过滤栏

快速过滤栏位于工具条下方，用于高频筛选操作。

## 4.1 快速过滤栏样式

```text
Ch:[全部 v]  Dir:[全部 v]  ID:[________]  Name:[________]  Data:[________]
DBC:[全部 v]  Frame:[全部 v]  Error:[全部 v]  搜索:[________]  [应用] [清除]
```

## 4.2 快速过滤字段

| 字段    | 控件   | 示例                    | 说明             |
| ----- | ---- | --------------------- | -------------- |
| Ch    | 下拉多选 | CAN1、CAN2             | 按通道过滤          |
| Dir   | 下拉多选 | Rx、Tx                 | 按方向过滤          |
| ID    | 输入框  | `0x180`、`0x100-0x1FF` | 按 ID 或 ID 范围过滤 |
| Name  | 输入框  | `BMS`                 | 按 DBC 报文名模糊匹配  |
| Data  | 输入框  | `01 02`               | 按 Data 内容匹配    |
| DBC   | 下拉框  | 已匹配、未匹配               | 按 DBC 匹配状态过滤   |
| Frame | 下拉框  | Data、Remote、Error     | 按帧类型过滤         |
| Error | 下拉框  | 全部、仅错误、排除错误           | 按错误状态过滤        |
| 搜索    | 输入框  | 任意关键字                 | 在当前显示结果中搜索     |

## 4.3 ID 输入规则

ID 输入框支持以下格式：

| 输入                  | 含义                |
| ------------------- | ----------------- |
| `0x180`             | 只显示 0x180         |
| `180`               | 按十六进制解释为 0x180    |
| `0x180,0x181,0x182` | 显示多个指定 ID         |
| `0x100-0x1FF`       | 显示 ID 范围          |
| `!0x180`            | 排除 0x180          |
| `0x180,!0x181`      | 显示 0x180，排除 0x181 |
| `STD:0x180`         | 只匹配标准帧 0x180      |
| `EXT:0x18FF50E5`    | 只匹配扩展帧 0x18FF50E5 |

## 4.4 Data 输入规则

Data 输入框支持以下格式：

| 输入                | 含义                         |
| ----------------- | -------------------------- |
| `01 02`           | Data 中包含连续字节 01 02         |
| `01 ?? 03`        | 第 1 字节为 01，第 3 字节为 03，中间任意 |
| `Data[0]=01`      | 第 0 字节等于 01                |
| `Data[2]>7F`      | 第 2 字节大于 0x7F              |
| `Data[0:2]=01 02` | 第 0 到 1 字节为 01 02          |
| `len=8`           | 数据长度为 8                    |
| `changed`         | 只显示 Data 有变化的报文            |

## 4.5 Name 输入规则

Name 支持模糊匹配：

| 输入        | 含义               |
| --------- | ---------------- |
| `BMS`     | 报文名包含 BMS        |
| `BMS*`    | 报文名以 BMS 开头      |
| `*Status` | 报文名以 Status 结尾   |
| `!Debug`  | 排除名称包含 Debug 的报文 |

## 4.6 快速过滤应用逻辑

- 用户输入后点击“应用”执行过滤；

- 按 Enter 也执行过滤；

- 点击“清除”清空所有快速过滤条件；

- 过滤后不删除原始缓存，只影响显示结果；

- 过滤状态应在状态栏显示；

- 过滤条件应可保存到工程文件；

- 过滤条件应可命名为“过滤预设”。

# 5. 高级过滤功能

## 5.1 高级过滤入口

高级过滤可通过以下入口打开：

- 报文窗口工具条“高级过滤”按钮；

- 快速过滤栏右侧“高级”按钮；

- 表格右键菜单“高级过滤”；

- 菜单栏：测量 → 报文过滤。

## 5.2 高级过滤弹窗布局

```text
┌────────────────────────────────────────────┐
│ 高级过滤                                    │
├────────────────────────────────────────────┤
│ 过滤规则列表                                │
│ [启用] [字段] [条件] [值] [包含/排除] [删除] │
├────────────────────────────────────────────┤
│ [+ 添加规则] [+ 添加规则组]                 │
├────────────────────────────────────────────┤
│ 规则组逻辑：全部满足 AND / 任一满足 OR        │
├────────────────────────────────────────────┤
│ [保存为预设] [加载预设] [应用] [取消]         │
└────────────────────────────────────────────┘
```

## 5.3 高级过滤字段

高级过滤支持以下字段：

| 字段           | 条件                               |
| ------------ | -------------------------------- |
| Channel      | 等于、不等于、属于列表                      |
| Direction    | Rx、Tx                            |
| CAN ID       | 等于、不等于、范围内、属于列表                  |
| Frame Format | 标准帧、扩展帧                          |
| Frame Type   | 数据帧、远程帧、错误帧                      |
| Protocol     | CAN、CAN FD                       |
| DLC          | 等于、大于、小于、范围内                     |
| Data Length  | 等于、大于、小于、范围内                     |
| Data Byte    | 指定字节等于/大于/小于                     |
| Data Pattern | 包含、通配符匹配、正则匹配                    |
| Message Name | 包含、等于、正则、不包含                     |
| Sender Node  | 包含、等于、不包含                        |
| Signal Name  | 包含、等于、正则                         |
| Signal Value | 大于、小于、等于、范围内                     |
| Cycle        | 大于、小于、范围内                        |
| Count        | 大于、小于                            |
| Error State  | Normal、Error、BusOff、ErrorPassive |
| DBC Match    | Matched、Unmatched                |
| Timestamp    | 时间范围内                            |
| Comment      | 包含、不包含                           |

## 5.4 过滤规则组

高级过滤必须支持规则组。

示例 1：只看 BMS 相关报文：

```text
Group Logic: AND
1. Channel == CAN1
2. MessageName contains BMS
3. Direction == Rx
```

示例 2：只看 BMS 电压异常：

```text
Group Logic: AND
1. MessageName contains BMS
2. SignalName == BMS_Voltage
3. SignalValue > 750
```

示例 3：排除调试报文：

```text
Exclude Group:
1. CAN ID in 0x700-0x7FF
2. MessageName contains Debug
```

## 5.5 过滤预设

用户可以保存常用过滤条件。

预设列表示例：

```text
BMS 报文
VCU 报文
错误帧
未匹配 DBC
高压相关信号
充电流程相关报文
```

每个预设保存内容：

- 预设名称；

- 过滤规则；

- 是否包含快速过滤条件；

- 是否包含列显示设置；

- 是否包含排序设置。

# 6. 搜索功能

搜索与过滤不同：过滤会改变显示集合，搜索只在当前显示集合中定位目标。

## 6.1 搜索栏

搜索栏可放在快速过滤栏右侧。

支持输入：

```text
0x180
BMS_SOC
01 02 03
Error
Timeout
```

## 6.2 搜索范围

搜索范围包括：

- CAN ID；

- 报文名称；

- Data；

- Comment；

- 信号名称；

- 信号物理值；

- 发送节点；

- 通道。

## 6.3 搜索结果行为

- 匹配行高亮；

- 支持上一个/下一个；

- 状态显示：`第 3 / 共 28 个匹配项`；

- 按 Enter 跳转下一个；

- Shift + Enter 跳转上一个；

- 搜索不影响过滤条件；

- 搜索结果可一键转为过滤条件。

# 7. 排序功能

## 7.1 表头排序

用户点击表头可排序。

支持排序字段：

| 字段      | 排序类型         |
| ------- | ------------ |
| No.     | 数值排序         |
| Time    | 时间排序         |
| Delta   | 数值排序         |
| Ch      | 字符串排序        |
| Dir     | Rx/Tx 排序     |
| ID      | 数值排序，不按字符串排序 |
| Name    | 字符串排序        |
| Type    | 字符串排序        |
| DLC     | 数值排序         |
| Len     | 数值排序         |
| Data    | 字节序列排序       |
| Cycle   | 数值排序         |
| Count   | 数值排序         |
| Comment | 字符串排序        |

## 7.2 排序交互

- 单击列头：升序；

- 再次点击：降序；

- 第三次点击：恢复默认时间顺序；

- Shift + 点击列头：添加多级排序；

- 列头显示排序箭头；

- 多级排序显示排序优先级数字，例如 `ID ↑ 1`、`Time ↓ 2`。

## 7.3 默认排序

Trace 模式默认按 Time 升序，即报文接收顺序。

Overwrite 模式默认按 ID 升序，也可切换为：

- 按最近更新时间排序；

- 按 Count 排序；

- 按 Cycle 排序；

- 按 Name 排序。

## 7.4 排序性能要求

- 排序只作用于当前显示数据集；

- 对 100,000 行数据排序应不造成界面长时间卡死；

- 排序过程应显示进度或忙碌状态；

- 实时接收状态下排序时，建议自动暂停 UI 刷新；

- 排序完成后再恢复刷新。

# 9. 分组显示功能

## 9.1 分组入口

在报文窗口顶部增加分组选项：

```text
分组：[无 v]
```

支持分组方式：

- 无分组；

- 按通道分组；

- 按方向分组；

- 按 CAN ID 分组；

- 按 DBC 报文名分组；

- 按发送节点分组；

- 按错误状态分组。

## 9.2 分组显示样式

按通道分组示例：

```text
▼ CAN1    Rx: 120000    Tx: 3000    Error: 0
    0x180    BMS_Status    01 02 03 04 05 06 07 08
    0x181    BMS_Voltage   00 10 20 30 40 50 60 70

▼ CAN2    Rx: 80000     Tx: 1000    Error: 2
    0x280    VCU_Status    11 22 33 44 55 66 77 88
```

按 ID 分组示例：

```text
▼ 0x180 BMS_Status    Count: 1200    AvgCycle: 100ms
    12.100000    Rx    01 02 03 04 05 06 07 08
    12.200000    Rx    01 02 03 04 05 06 07 09
```

## 9.3 分组操作

- 分组可展开/折叠；

- 支持全部展开；

- 支持全部折叠；

- 分组标题显示统计信息；

- 分组状态保存到工程文件；

- 分组下仍支持过滤和搜索。

# 12. 筛选、过滤、搜索、排序的关系

必须明确以下逻辑：

## 12.1 数据处理顺序

显示数据处理顺序为：

```text
原始报文缓存
    ↓
基础显示模式 Trace / Overwrite
    ↓
快速过滤
    ↓
高级过滤
    ↓
列筛选
    ↓
搜索高亮
    ↓
排序
    ↓
分组
    ↓
UI 显示
```

## 12.2 概念区别

| 功能  | 是否减少显示数据 | 是否改变顺序 | 主要用途      |
| --- | -------- | ------ | --------- |
| 过滤  | 是        | 否      | 只看符合条件的数据 |
| 筛选  | 是        | 否      | 按某一列选择值   |
| 搜索  | 否        | 否      | 定位目标      |
| 排序  | 否        | 是      | 改变显示顺序    |
| 分组  | 否        | 是      | 按类别组织显示   |

## 12.3 实时接收时的处理原则

实时接收过程中：

- 过滤条件应对新报文实时生效；

- 搜索结果应随新报文动态更新；

- 排序开启时，不建议每帧重新排序；

- 排序状态下 UI 应按批次刷新；

- Trace 模式下如果用户手动排序，自动滚动应暂停；

- Overwrite 模式更适合实时排序和分组。

# 13. 高性能实现要求

## 13.1 缓存设计

CAN 报文显示窗口至少维护三层数据：

```text
RawMessageBuffer       原始报文环形缓存
DisplayMessageBuffer   过滤后的显示缓存
VisibleRows            当前 UI 虚拟滚动可见行
```

## 13.2 UI 刷新要求

- 接收线程不能直接操作 UI；

- 报文进入线程安全队列；

- UI 定时批量拉取数据；

- 默认 UI 刷新周期 100ms；

- 高负载时允许降低刷新频率；

- 表格必须支持虚拟滚动；

- 不能将百万级数据直接绑定到普通 ObservableCollection。

## 13.3 大数据量要求

| 场景     | 要求             |
| ------ | -------------- |
| 10 万行  | 正常滚动、排序、过滤     |
| 100 万行 | 可以过滤、搜索，允许短暂等待 |
| 高速接收   | 不丢接收缓存，不阻塞 UI  |
| 过滤变化   | 应显示忙碌状态        |
| 清空窗口   | 1 秒内完成         |

## 13.4 推荐实现方式

- 表格使用支持虚拟化的 DataGrid；

- 内部使用分页或虚拟数据源；

- 过滤、排序在后台线程执行；

- UI 只渲染可见行；

- Data 字节高亮使用自定义 CellTemplate；

- 对 ID、Name、Channel 建立索引，提高过滤速度。

# 15. 给 AI 开发工具的补充提示词

请重点细化并实现中央主区域的 CAN 报文显示窗口。

该窗口必须是专业 CAN 工具风格的高性能表格，不是简单 ListView。

窗口上方需要有快捷工具条和快速过滤栏。快捷工具条包含 Trace 模式、Overwrite 模式、暂停显示、跟随最新、清空、快照、导出、列设置、高级过滤。快速过滤栏包含通道、方向、ID、Name、Data、DBC 匹配状态、帧类型、错误状态和搜索输入框。

CAN 报文表格必须支持排序、筛选、过滤、搜索、分组和右键菜单。点击表头可以排序，Shift + 点击表头可以多级排序。每列需要支持类似 Excel 的下拉筛选。快速过滤支持 ID 单值、多值、范围、排除语法，例如 0x180、0x180,0x181、0x100-0x1FF、!0x180。Data 过滤支持连续字节、通配符和指定字节位置，例如 01 02、01 ?? 03、Data[0]=01、Data[2]>7F。

搜索功能不能改变当前显示集合，只负责高亮和跳转匹配行。过滤和筛选会减少显示数据，排序会改变显示顺序，分组会按通道、方向、ID、报文名、发送节点或错误状态组织数据。

Trace 模式默认按时间顺序追加显示。Overwrite 模式相同通道、方向和 CAN ID 只显示一行，收到新帧后刷新该行，Count 累加，Delta 更新，Data 变化字节高亮。实时接收过程中，过滤条件必须对新报文实时生效；排序开启时不能每帧排序，应批量刷新；接收线程不能直接更新 UI。

请使用虚拟滚动或虚拟数据源实现高性能显示。内部至少区分 RawMessageBuffer、DisplayMessageBuffer、VisibleRows。10 万行数据排序和过滤不能卡死 UI。Data 字段需要使用等宽字体，并支持变化字节局部高亮。
